# Приморская картинная галерея — 3D-сайт

Интерактивный 3D-сайт с описанием **29 картин** коллекции. Посетитель сканирует
QR-код под картиной → попадает на страницу с рассказом об этой работе.

## Содержимое
```
site/
├── index.html        ← открыть этот файл (точка входа)
├── main.js           ← 3D-сцена, карусель, логика
├── data.js           ← тексты и метаданные всех 29 картин
├── qr_data.js        ← QR-коды, встроенные в сайт (для панели)
├── lib/three.min.js  ← Three.js (локально, работает офлайн)
└── qrcodes/          ← 29 готовых PNG QR-кодов для печати под картинами
    ├── qr_01.png … qr_29.png
```

## Как это работает
- Каждый QR ведёт на `https://primgallery.com/#p=N`, где `N` — номер картины (1–29).
- При открытии ссылки с `#p=N` сайт автоматически прокручивает карусель
  к нужной картине и открывает её описание.
- 3D-карусель: тяните мышью/пальцем для вращения, колесо — масштаб,
  стрелки ← →, клик по картине — открыть описание, кнопка «Все 29 работ» — сетка.

## Управление
| Действие | Как |
|---|---|
| Вращать галерею | перетаскивание / стрелки ‹ › / клавиши ← → |
| Картину в центр | клик по боковой картине |
| Открыть описание | клик по центральной картине / «Подробнее» / Enter |
| Масштаб | колесо мыши / пинч двумя пальцами |
| Авто-показ (киоск) | кнопка «Авто-показ» / пробел |
| Все работы списком | кнопка «Все 29 работ ▦» |
| Закрыть | × / Esc |

> **Авто-показ** удобен для размещения на экране-стенде в самой галерее:
> картины сменяются автоматически каждые ~4.5 сек; любое действие пользователя
> его приостанавливает.

## Печать QR-кодов
Файлы в `qrcodes/` уже оформлены (рамка + номер + название) и готовы к печати —
разместите каждый под соответствующей картиной.

## Перегенерация QR (если изменится адрес сайта)
Замените `BASE` в `gen_qr.py` и `gen_qr_data.py`, затем:
```bash
pip install "qrcode[pil]"
python3 gen_qr.py        # → site/qrcodes/*.png (для печати)
python3 gen_qr_data.py   # → site/qr_data.js  (встроенные в сайт)
```

## Деплой
Загрузите содержимое папки `site/` на хостинг так, чтобы `index.html`
открывался по адресу `https://primgallery.com/`. Никакой сборки не требуется —
это статический сайт.

## О изображениях
Полотна в 3D-сцене — авторские процедурные абстракции, сгенерированные в
исторической цветовой палитре каждой работы (это художественная заставка,
а не репродукция оригинала). При желании их можно заменить на реальные
фотографии картин: положите файлы и подставьте их в `main.js`
вместо `makeArtCanvas`.

## Как добавить фотографии и звук
- Положите фотографии картин в папку `site/images/` именованные `01.jpg`…`29.jpg`.
- В `data.js` можно добавить поле `image: 'images/01.jpg'` у нужной записи (или
  просто положить файлы с такими именами — скрипт подхватит их автоматически).
- Для фоновой музыки положите файл `hall.mp3` в `site/audio/` и при необходимости
  замените путь в `main.js` (по умолчанию `audio/hall.mp3`).

> Примечание: в текущем наборе `site/images/` найдены реальные фото для картин
> 1–28. Для №13 (`Репин — Портрет М. Н. Галкина-Враского`) и №29 (`Голландия XVII в. — Натюрморт`)
> реальных JPG-файлов в пакете нет — вместо них созданы временные плейсхолдеры `13.jpg`
> и `29.jpg`.
